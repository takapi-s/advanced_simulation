計算時間のスケーリング解析

行列積の浮動小数点演算回数は O(n³) である。そこで，計算時間 T が

T ∝ nᵖ

と仮定し，両辺の対数をとって

log T = p log n + c

の形で最小二乗法による回帰を行い，指数 p を推定した。

項目 値
推定された指数 p 2.91
決定係数 R² 0.9999

理論値 p = 3 に対し，実測値は p ≈ 2.91 であった。MKL の高度に最適化された DGEMM 実装や，小さい n におけるメモリ確保・初期化などの定数項の影響により，わずかに 3 より小さい値となったと考えられる。いずれにせよ n が増大するにつれ計算時間がおおよそ 3 乗で増加する傾向が確認できた。

参考として，隣接する測定点の比から見た局所的な指数を示す。局所指数は

p_local = log(T₂ / T₁) / log(n₂ / n₁)

で定義した。

n₁ → n₂ n₂ / n₁ T₂ / T₁ 局所指数 p_local
2000 → 4000 2.0 7.24 2.86
4000 → 6000 1.5 3.27 2.93
6000 → 8000 1.33 2.33 2.96
8000 → 10000 1.25 1.97 2.97

n が大きくなるほど局所指数は 3 に近づいており，漸近的に O(n³) のスケーリングに従うことがわかる。

スケーリング解析

平均計算時間に対して log T = p log n + c の最小二乗回帰を行い，指数 p を推定した。

ライブラリ 推定指数 p 決定係数 R²
rocBLAS（AMD MI210） 2.57 0.989
cuBLAS（NVIDIA A100） 2.60 0.999

理論的な演算量は O(n³) である。これに対し，実測の指数はいずれも p ≈ 2.6 となった。計測時間にはホストとデバイス間のデータ転送 O(n²) が含まれているため，特に n が小さいときは転送コストの影響が大きく，全体の指数 p が 3 より小さく見える。n = 12000 以降では隣接点の局所指数が 2.8〜3.0 付近に近づき，計算本体の O(n³) スケーリングが支配的になっている。

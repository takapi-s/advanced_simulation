## 【1】(1) 共役勾配法による連立一次方程式の求解

与えられた 5 次正方行列 **A** と右辺ベクトル **b** に対し，連立一次方程式 **Ax = b** を共役勾配法（CG 法）で解いた。

### 設定

| 項目 | 値 |
|------|-----|
| 行列のサイズ | 5 |
| 初期値 **x**⁽⁰⁾ | (0, 0, 0, 0, 0)ᵀ |
| 収束判定 | ‖**r**‖₂ &lt; 1.0 × 10⁻¹²（**r** = **b** − **Ax**） |
| 最大反復回数 | 1000 |

### 結果

| 項目 | 値 |
|------|-----|
| 収束回数 | 5 回 |
| 最終残差 ‖**r**‖₂ | 1.89 × 10⁻¹⁴ |

解ベクトル **x**:

| 成分 | 値 |
|------|-----|
| x₀ | 2.0000000000 |
| x₁ | 1.0000000000 |
| x₂ | 5.0000000000 |
| x₃ | 3.0000000000 |
| x₄ | 4.0000000000 |

すなわち **x** = (2, 1, 5, 3, 4)ᵀ である。残差ベクトル **b** − **Ax** の各成分は 10⁻¹⁴ 程度以下であり，正しい解が得られた。

## 【2】行列積 **B** × **B** の計算時間測定

n × n の行列 **B** の各要素に [0.0, 1.0] の一様乱数を代入し，行列積 **C** = **B** × **B** を Intel MKL の `cblas_dgemm` で計算した。数値型は `double` とした。

### 測定条件

| 項目 | 値 |
|------|-----|
| 測定する時間 | **wall clock time**（`CLOCK_MONOTONIC`） |
| 各 n の試行回数 | 5 回 |
| ウォームアップ | 各 n について計測前に 1 回 DGEMM を実行 |
| 測定ホスト | ydev06.edu.tut.ac.jp（窓口サーバ） |

### 計測結果（秒）

| n | trial 1 | trial 2 | trial 3 | trial 4 | trial 5 | 平均 |
|---|---------|---------|---------|---------|---------|------|
| 2000 | 0.034112 | 0.034037 | 0.034242 | 0.032752 | 0.034112 | **0.033851** |
| 4000 | 0.243081 | 0.240849 | 0.266376 | 0.237163 | 0.237745 | **0.245043** |
| 6000 | 0.813849 | 0.797001 | 0.798413 | 0.812872 | 0.789449 | **0.802317** |
| 8000 | 1.874267 | 1.864863 | 1.876403 | 1.862097 | 1.865582 | **1.868642** |
| 10000 | 3.696918 | 3.652473 | 3.684306 | 3.692781 | 3.682202 | **3.681736** |

### 計算時間のスケーリング解析

行列積の浮動小数点演算回数は O(n³) である。そこで，計算時間 T が

　T ∝ nᵖ

と仮定し，両辺の対数をとって

　log T = p log n + c

の形で最小二乗法による回帰を行い，指数 p を推定した。

| 項目 | 値 |
|------|-----|
| 推定された指数 p | **2.91** |
| 決定係数 R² | 0.9999 |

理論値 p = 3 に対し，実測値は p ≈ 2.91 であった。MKL の高度に最適化された DGEMM 実装や，小さい n におけるメモリ確保・初期化などの定数項の影響により，わずかに 3 より小さい値となったと考えられる。いずれにせよ n が増大するにつれ計算時間がおおよそ 3 乗で増加する傾向が確認できた。

参考として，隣接する測定点の比から見た局所的な指数を示す。局所指数は

　p_local = log(T₂ / T₁) / log(n₂ / n₁)

で定義した。

| n₁ → n₂ | n₂ / n₁ | T₂ / T₁ | 局所指数 p_local |
|---------|---------|---------|------------------|
| 2000 → 4000 | 2.0 | 7.24 | 2.86 |
| 4000 → 6000 | 1.5 | 3.27 | 2.93 |
| 6000 → 8000 | 1.33 | 2.33 | 2.96 |
| 8000 → 10000 | 1.25 | 1.97 | 2.97 |

n が大きくなるほど局所指数は 3 に近づいており，漸近的に O(n³) のスケーリングに従うことがわかる。

### スケーリング解析

平均計算時間に対して log T = p log n + c の最小二乗回帰を行い，指数 p を推定した。

| ライブラリ | 推定指数 p | 決定係数 R² |
|------------|-----------|-------------|
| rocBLAS（AMD MI210） | **2.57** | 0.989 |
| cuBLAS（NVIDIA A100） | **2.60** | 0.999 |

理論的な演算量は O(n³) である。これに対し，実測の指数はいずれも p ≈ 2.6 となった。計測時間にはホストとデバイス間のデータ転送 O(n²) が含まれているため，特に n が小さいときは転送コストの影響が大きく，全体の指数 p が 3 より小さく見える。n = 12000 以降では隣接点の局所指数が 2.8〜3.0 付近に近づき，計算本体の O(n³) スケーリングが支配的になっている。

### CPU（MKL）との比較（参考）

同一の行列サイズ n における平均計算時間を比較する（【2】の MKL 結果は n = 4000, 8000 のみ重複）。

| n | MKL DGEMM（CPU） | rocBLAS（MI210） | cuBLAS（A100） |
|---|------------------|------------------|----------------|
| 4000 | 0.245 sec | 0.014 sec | 0.019 sec |
| 8000 | 1.869 sec | 0.065 sec | 0.103 sec |

GPU 版はいずれも CPU 版より高速であり，特に rocBLAS / MI210 では n = 8000 で約 29 倍の高速化が得られた。
